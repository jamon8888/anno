# Infrastructure Gaps Analysis

This document identifies disconnections and integration opportunities across Anno's
codebase. It serves as a roadmap for unification.

## Module Landscape

| Module | Lines | Purpose | Integration Status |
|--------|-------|---------|-------------------|
| `joint/` | ~6K | Durrett & Klein joint model | Partially connected |
| `box_embeddings.rs` | ~3K | Geometric coref | Standalone training |
| `cdcr.rs` | ~1.6K | Cross-document coref | Basic string similarity |
| `cluster_encoder.rs` | ~1.7K | xCoRe-style cluster merging | Scaffolding only |
| `cross_context.rs` | ~680 | Within→cross-context bridge | Not wired to CDCR |
| `mention_ranking.rs` | ~670 | Mention-ranking coref | CoreferenceResolver (new) |

## Disconnected Systems

### 1. Multiple Coreference Approaches

```
MentionRankingCoref ─────────────┐
SimpleCorefResolver ─────────────┼──→ CoreferenceResolver trait
BoxCorefResolver (commented) ────┘
       │
       │   (no connection)
       ↓
JointModel.resolve() ← Durrett-Klein joint inference
       │
       │   (no connection)
       ↓
CDCRResolver ← cross-document clustering
```

**Gap**: Three systems solve related problems but don't share:
- Mention representations
- Scoring functions
- Cluster data structures

**Fix**: Create `MentionEmbedding` trait that all systems can produce/consume.

### 2. Training vs Inference Split

| Component | Has Training | Has Inference | Weights Format |
|-----------|--------------|---------------|----------------|
| `joint/learning.rs` | ✅ | ✅ | `JointWeights` (serde) |
| `box_embeddings.rs` | ✅ | ✅ | `BoxConfig` (inline) |
| `mention_ranking.rs` | ❌ | ✅ | Hardcoded |
| `GLiNER` | ❌ | ✅ | ONNX external |
| `NuNER` | ❌ | ✅ | ONNX external |

**Gap**: No unified weight loading/saving format. `JointWeights` uses serde but
isn't exposed via CLI. Box embeddings have their own format.

**Fix**: Implement `ModelWeights` trait with `save()` / `load()` for all trainable
components. Add `anno weights export` CLI command.

### 3. ClusterEncoder Not Used

`cluster_encoder.rs` and `neural_cluster_encoder.rs` implement xCoRe-style
incremental cluster merging but are never called from:
- `cdcr.rs` (uses simple string matching)
- `coalesce/` crate (unknown, needs investigation)
- Any CLI command

**Gap**: 3,000+ lines of sophisticated clustering code with no entry point.

**Fix**: Wire `CDCRResolver` to use `NeuralClusterEncoder` when available.

### 4. Cross-Context Bridge Incomplete

`cross_context.rs` defines:
```rust
pub struct CrossContextBridge {
    within_doc_resolver: Arc<dyn CoreferenceResolver>,
    cross_doc_resolver: Arc<dyn CrossDocResolver>,  // ← trait doesn't exist
}
```

**Gap**: The `CrossDocResolver` trait is used but never defined or implemented.

**Fix**: Define trait in `cdcr.rs`, implement for `CDCRResolver`.

### 5. Dataset Registry Macro Mismatch

The `define_datasets!` macro supports `alt_sources` field but many entries use it
in wrong positions relative to `categories`. This causes compilation failures.

Current field order in macro:
```
... hf_config, alt_sources, s3_path, version, annotation_quality, train_size, dev_size, test_size, categories
```

**Gap**: ~160 entries have `alt_sources` but macro parsing is order-sensitive.

**Fix**: Either enforce field order in all entries or make macro order-independent.

### 6. Loader Expects Missing Methods

`loader.rs` calls `DatasetId::is_african_language()` but this method isn't
generated by the `define_datasets!` macro.

**Gap**: Code references methods that don't exist.

**Fix**: Either add category-based method generation to macro, or remove calls.

## Recommended Integration Order

1. **Fix compilation** (dataset_registry.rs, loader.rs)
   - Remove or fix `alt_sources` usage
   - Add missing `is_african_language()` method

2. **Unify coref output** (~2 days)
   - `MentionRankingCoref` already implements `CoreferenceResolver` (done)
   - Make `JointModel` return `Vec<CorefChain>`
   - Add `CoreferenceResolver` impl for `CDCRResolver`

3. **Connect CDCR → ClusterEncoder** (~1 day)
   - Create `CrossDocResolver` trait
   - `CDCRResolver` delegates to `ClusterEncoder` for merge decisions

4. **Unified weight format** (~2 days)
   - `ModelWeights` trait
   - CLI commands for export/import

5. **Pipeline documentation** (~1 day)
   - Update ARCHITECTURE.md with actual data flow
   - Document what's connected vs standalone

## Testing Gaps

**Fix**: Add `tests/coref_pipeline_integration.rs` testing:
```rust
#[test]
fn within_doc_to_cross_doc_flow() {
    let within = MentionRankingCoref::new();
    let cross = CDCRResolver::new(...);
    // Verify data flows correctly between systems
}
```

## Serving Infrastructure

**Status**: None.

Anno has no HTTP server, gRPC service, or batch prediction CLI.

Recommendation: Start minimal with `anno serve --port 8080` that:
1. Loads one NER backend
2. Accepts JSON via POST
3. Returns entities as JSON

Use `axum` for simplicity. ~200 lines.
