# anno-rag v0.1 — Walking Skeleton Spec

**Date:** 2026-05-12
**Status:** ready for implementation
**Goal:** Ship a working end-to-end loop on real French documents in ~1 week.

The full v1+ design is archived at [2026-05-12-anno-rag-design.md](./2026-05-12-anno-rag-design.md). This v0.1 is the *minimum* that proves the pipeline works on real data; everything else is iteration on top.

---

## 1. What v0.1 does

A single Rust binary `anno-rag` with two subcommands:

```sh
anno-rag ingest /path/to/folder [--recursive] [--output /path/to/output]
anno-rag search "what is the termination clause" [--top-k 10]
```

`ingest`:
1. Walks the folder (recursive into subfolders if `--recursive`)
2. For each file: kreuzberg → text + chunks
3. Detects French PII: 4 regex (NIR, SIRET, IBAN-FR, FR phone) + anno `gliner_pii` for names/addresses
4. Pseudonymizes via cloakpipe SqliteVault — original text and tokens never co-exist outside the vault
5. Embeds chunks (candle BGE-small multilingual)
6. Stores in LanceDB
7. Writes `output/<doc>.anon.md` (pseudonymized copy)

`search`:
1. Pseudonymizes the query
2. LanceDB hybrid search (native FTS + vector via `RRFReranker`)
3. Returns top-K chunks with `{score, source_path, page, char_offsets, pseudonymized_text}`

That's it.

## 2. What v0.1 does NOT do

All of the following exist in [the v1 design spec](./2026-05-12-anno-rag-design.md) and are explicitly deferred:

- MCP server (v0.2)
- HTTP API (v0.2)
- GDPR audit log / Art. 30 register / Art. 17 erasure / Art. 15 access (v0.3)
- Packs / LoRA hot-swap / vertical profiles (v0.4)
- Citation graph (v0.5)
- Privilege gating (v0.5)
- Tabular review (v1.1 — plan at [2026-05-12-anno-rag-tabular-review-v1.1.md](../plans/2026-05-12-anno-rag-tabular-review-v1.1.md))
- Multi-crate workspace decomposition (when `cargo build` exceeds 30s)
- Cross-encoder rerank (when we have an eval set to justify it)
- Hierarchical structure typing (when generic chunking proves inadequate)
- Cross-platform code signing (when we're ready to distribute)
- Process-safety lock file (when v0.2 adds a daemon)

## 3. Single crate, single binary

```
crates/anno-rag/
├── Cargo.toml
└── src/
    ├── main.rs        # clap, dispatches to ingest::run / search::run
    ├── ingest.rs      # kreuzberg → text → chunks
    ├── detect.rs      # anno gliner_pii + FR regex set
    ├── vault.rs       # cloakpipe SqliteVault wrapper
    ├── embed.rs       # candle BGE-small bridge
    ├── store.rs       # lancedb: open, upsert, hybrid_search
    └── pipeline.rs    # the ingest pipeline: extract → detect → pseudonymize → embed → store
```

**One crate, no traits, no abstractions.** When the codebase actually needs splitting, split it. Not before.

## 5. Data flow

```
folder/file.pdf
   │ kreuzberg::extract_file (sync API, in tokio::spawn_blocking)
   │   ExtractionConfig { chunking: Some(MarkdownAware { max_chars: 2048, max_overlap: 256 }),
   │                      ocr: Some(Tesseract { language: "fra" }),
   │                      pages: Some(extract_pages: true) }
text + chunks[] + pages[]
   │
   │ detect::scan
   │   - cloakpipe PatternDetector (FR regex pack: NIR / SIRET / IBAN-FR / phone-FR)
   │   - anno::StackedNER with gliner_pii backend (PER, ORG, LOC, plus 60+ PII categories)
   │   - merge + dedup spans (helper from cloakpipe detector/mod.rs:160)
entities[]
   │
   │ vault::pseudonymize
   │   - cloakpipe SqliteVault (AES-256-GCM per-row, WAL)
   │   - 32-byte key derived from OS keyring secret via Argon2id
   │   - cloakpipe Replacer: deterministic tokens within session (PER_1, NIR_3, ...)
pseudonymized text + token mappings persisted
   │
   │ embed::vectorize
   │   - candle BGE-small-multilingual (intfloat/multilingual-e5-small at ~118M params, ~470 MB)
   │   - per-chunk f32×384 vector
chunk_embeddings[]
   │
   │ store::upsert
   │   - LanceDB table `chunks`
   │   - merge_insert(["doc_id", "chunk_idx"]) — idempotent re-ingest
   │   - IVF_HNSW_SQ index built lazily when chunk count exceeds 10000
   │
   ▼
output/<doc>.anon.md   (pseudonymized copy on disk)
~/.anno-rag/index.lance/   (searchable index)
~/.anno-rag/vault.db        (encrypted token mappings)
```

For `search`:

```
query text
   │ vault::pseudonymize_query (session-scoped — query might contain a name)
   │ embed::vectorize (single vector)
   │ store::hybrid_search:
   │   tbl.query()
   │      .full_text_search(FullTextSearchQuery::new(q))     ← native BM25 via FTS index
   │      .nearest_to(q_vec)                                  ← vector
   │      .limit(K * 5)
   │      .rerank(RRFReranker::default())                     ← native RRF k=60
   │
   ▼
top-K [{score, doc_path, page, char_offsets, pseudonymized_chunk_text}]
```

## 6. Schema (LanceDB `chunks` table)

```rust
Field::new("chunk_id",          DataType::FixedSizeBinary(16), false),  // UUID v7
Field::new("doc_id",            DataType::FixedSizeBinary(16), false),  // UUID v7
Field::new("source_path",       DataType::Utf8,                false),
Field::new("folder_path",       DataType::Utf8,                false),  // F1 from architecture review
Field::new("chunk_idx",         DataType::UInt32,              false),
Field::new("text_pseudo",       DataType::Utf8,                false),  // already pseudonymized
Field::new("page",              DataType::UInt32,              true),
Field::new("char_start",        DataType::UInt32,              true),
Field::new("char_end",          DataType::UInt32,              true),
Field::new("text_hash",         DataType::FixedSizeBinary(32), false),  // sha256(text_pseudo)
Field::new("vector",            DataType::FixedSizeList(Float32, 384), false),
Field::new("ingested_at",       DataType::Timestamp(Microsecond, None), false),
```

That's the whole schema. No structural metadata, no citation columns, no privilege column — those land in v0.3+.

## 7. French PII regex pack (v0.1 minimal)

Built into `detect.rs` directly (no pack format yet):

- **NIR (Sécurité Sociale)**: `\b[12]\d{2}(0[1-9]|1[0-2])(2[AB]|\d{2})\d{3}\d{3}\d{2}\b`
- **SIRET**: `\b\d{14}\b` (with Luhn check in code)
- **SIREN**: `\b\d{9}\b` (with Luhn check)
- **IBAN-FR**: `\bFR\d{2}\s?(?:\d{4}\s?){5}\d{3}\b`
- **FR phone**: `\b(?:\+33[\s\.-]?|0)[1-9](?:[\s\.-]?\d{2}){4}\b`

Names / addresses / organizations come from `anno::StackedNER` with `gliner_pii` backend (already in v1.0 anno).

## 8. Cross-platform packaging

`cargo install --path crates/anno-rag` on each target. No installer framework. The user runs:

```sh
# macOS
cargo install --path crates/anno-rag

# Windows
cargo install --path crates/anno-rag
```

That's the distribution for v0.1. Real packaging (`.dmg` / `.msi` / code signing) lands in v0.2 once the binary is stable.

## 9. Acceptance criteria

v0.1 ships when **all** of these pass on a clean dev machine:

1. `cargo build --release -p anno-rag` succeeds on macOS (M-series) and Windows (x86_64)
2. `cargo test -p anno-rag` passes — at minimum: ingest of 3 fixture French docs, search returns the expected passage
3. `anno-rag ingest testdata/legal-fr/` runs end-to-end on a real folder of 5–10 mixed PDFs/DOCX
4. `anno-rag search "résiliation"` returns relevant chunks ranked by hybrid score
5. The output `*.anon.md` files contain pseudonymized tokens (no raw PII visible)
6. The vault file (`~/.anno-rag/vault.db`) exists and is encrypted (cannot be opened as plaintext SQLite)
7. Total time from `cargo build` to first search result: under 5 minutes on a warm cache, under 15 minutes on a cold cache

## 10. Out-of-scope (v0.2+)

Everything listed in §2 plus:
- Configuration via TOML (v0.1 uses CLI flags only)
- Hot-swappable backends (v0.1 hardcodes BGE-small-multilingual)
- Concurrency tuning (v0.1 uses `tokio` defaults)
- Telemetry / metrics (v0.1 uses `tracing` for stdout only)
- Error recovery on partial ingest failure (v0.1: bail on first error)
- Re-ingest idempotency beyond `merge_insert` (v0.1 doesn't detect content drift)
- Internationalization (v0.1 is FR-default; EN works because BGE is multilingual)

## 11. Path to v0.2+

After v0.1 ships and runs on a real folder:
