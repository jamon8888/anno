# anno-rag v0.5 — Performance Budget Spec

**Date:** 2026-05-13
**Author:** brainstorming session (Claude + jamon)
**Status:** approved design, awaiting implementation plan
**Target release:** v0.5.0
**Depends on:** v0.3 #001 (GLiNER2Fastino multi-v1) merged

## 1. Why this spec exists

The v0.2 bench on `piighost-test-multi-format` (see `crates/anno-rag/docs/dev-notes/bench-v0.2-piighost-test.md`) measured **3.95 GB peak RSS** ingesting 6 small French documents. The product's stated target user is a French independent lawyer / small accounting firm on a **Windows laptop with 8 GB RAM** (per v1 Northstar design §3). 3.95 GB peak makes that deployment scenario unworkable: Windows idle (~3 GB) + Word + Chrome + Claude Desktop saturates the box before anno-rag opens its first PDF.

Cold-start latency, search latency, and ingest throughput were also never measured under release-mode + warm-path conditions. The team has been operating on optimistic assumptions.

**v0.5 fixes both: it imposes a measured, CI-enforced performance budget on RAM, cold-start, and search latency, and gives the codebase the four memory reductions and one bench harness needed to hit that budget.**

## 2. Goals and non-goals

### Goals

1. **Hard SLO** on a laptop-class profile (8 GB / 4 cores Windows):
   - Peak RSS idle MCP server: < 200 MB
   - Peak RSS after 1 search (warm): < 1.5 GB
   - Cold-start (`anno-rag mcp` to ready): < 2s
   - First search latency (cold model load amortized): < 8s
   - Search p95 hot-path: < 200ms
   - Ingest throughput release-mode: < 10s/doc on the e2e fixture corpus
2. **Recall preservation:** recall@10 stays ≥ 95% of the v0.2 fp32 baseline after fp16 quantization.
3. **Regression prevention:** criterion-based bench harness + CI gate (>15% regression fails the PR).
4. **Observable budget:** `anno-rag bench` subcommand any user / dev can run locally to reproduce the SLO measurements on their own corpus.

### Non-goals

- **Format coverage** (XLSX / CSV / TSV / JSONL — issue #028) — separate v0.6 spec.
- **int8 / int4 quantization** of the embedder. Defer until we see whether fp16 alone suffices.
- **GPU acceleration.** Target hardware has no GPU; CUDA / Metal paths stay best-effort.
- **LanceDB IVF tuning.** Default `IVF_HNSW_SQ` parameters are good enough at the corpus sizes v0.5 targets.
- **Encryption at rest** (#019) — v1.0.
- **Process-safety lock file** (#020) — v1.0.

## 3. Architecture

Five workstreams. Each lands as an independently mergeable task with measurable RSS / latency impact.

### 3.1 Bench harness (#029) — lands FIRST

`crates/anno-rag/benches/` (criterion 0.5):

| Bench | Measures | Hard gate |
| --- | --- | --- |
| `bench_cold_start.rs` | `Pipeline::new` + `vault_stats` round-trip | p95 < 2s |
| `bench_idle_rss.rs` | `sysinfo::Process::memory()` after Pipeline init | < 200 MB |
| `bench_search.rs` | 100× `pipeline.search(q, 10)` warm | p95 < 200ms |
| `bench_ingest.rs` | Ingest 3-doc e2e fixture corpus, release build | < 30s wall |
| `bench_recall.rs` | recall@10 over 10 reference queries | ≥ 95% of `main` baseline |

**Portability:** RSS measurement via `sysinfo = "0.32"` (Linux + macOS + Windows). Each OS has its own baseline file because "RSS" semantics differ (Linux RSS vs Windows WorkingSet).

**CI integration:** `.github/workflows/bench.yml` runs on every PR against `main`, uses `critcmp` to compare; >15% regression fails the job. Nightly job on `main` refreshes the baseline artifact.

**User-facing:** `anno-rag bench --corpus <dir>` subcommand emits a markdown report (cold-start, idle RSS, search p50/p95/p99, ingest throughput, peak RSS). Same numbers as CI but on the user's own data.

### 3.2 Lazy-init Embedder & Detector (#023)

Currently `Pipeline::new` eagerly loads ~2 GB of model weights (embedder + StackedNER stack). For a user who only ever calls `vault_stats` or `rehydrate`, that's pure waste.

```rust
pub struct Pipeline {
    embedder: tokio::sync::OnceCell<Arc<Embedder>>,
    detector: tokio::sync::OnceCell<Arc<Detector>>,
    store: Store,            // lightweight, eager
    vault: Vault,            // lightweight, eager
    config: AnnoRagConfig,
}

impl Pipeline {
    async fn embedder(&self) -> Result<&Arc<Embedder>> {
        self.embedder
            .get_or_try_init(|| async {
                Embedder::new(&self.config).await.map(Arc::new)
            })
            .await
    }
    // ditto detector()
}
```

All call sites (`ingest`, `search`, `detect`) go through the accessor. Idle MCP server = tokio runtime + LanceDB connection + vault FD ≈ <200 MB.

**Trade-off:** first `search` call pays the model load (~3-5s). Acceptable per SLO (first search < 8s amortized).

### 3.3 Drop bundled Tesseract, OCR opt-in (#024)

`kreuzberg`'s `ocr` feature statically links libtesseract + leptonica + bundled tessdata for fra/eng (~500 MB resident, even if no PDF needs OCR).

**Cargo.toml:**
```toml
kreuzberg = { version = "=4.9.7", features = ["pdf","office","html","tokio-runtime"] }  # no "ocr"
```

**Runtime:** OCR is now opt-in via the `ocr` Cargo feature **and** a runtime `--enable-ocr` CLI flag. When enabled, post-extraction logic detects empty text layer and forks system `tesseract`:

```rust
if config.enable_ocr && extracted.text.trim().is_empty() && doc_is_pdf(&path) {
    match tokio::process::Command::new("tesseract")
        .arg(&path)
        .arg("-")
        .arg("-l").arg("fra+eng")
        .output()
        .await
    {
        Ok(out) if out.status.success() => /* use out.stdout */,
        Ok(_) | Err(_) => warn!(path = ?path, "OCR fork failed; document text empty"),
    }
}
```

Timeout 60s per page. README documents `winget install tesseract-ocr.tesseract-ocr` / `brew install tesseract` / `apt install tesseract-ocr tesseract-ocr-fra`.

**Default behavior:** silent skip with structured warning. Lawyers using Word→PDF or LibreOffice→PDF always have a text layer; the OCR path is for scanned docs only, which is the exception not the rule.

### 3.4 Single-backend NER, post v0.3 #001 (#025)

`detect.rs` line 91 currently has a `TODO(v0.3 #1)` marker. With v0.3 #001 merged, replace:

```rust
// before
let stacked = StackedNER::default();

// after
let model = GLiNER2Fastino::from_pretrained("SemplificaAI/gliner2-multi-v1-onnx").await?;
```

Drop `bert-base-NER` + `gliner_small-v2.1` + `NuNER_Zero` from `warmup_model.rs` candidate fallback chain — only the multi-v1 ONNX is downloaded.

**Memory impact:** ~900 MB saved (the three fallback backends are no longer co-resident).

### 3.5 Embedder fp16 on CPU (#026)

candle 0.10 supports `DType::F16` on CPU via softfloat. Load `multilingual-e5-small` with F16 weights, cast inputs to F16 before forward pass.

```rust
let dtype = match config.embedder_dtype.as_deref() {
    Some("f32") => DType::F32,
    _ => DType::F16,  // default v0.5
};
let vb = VarBuilder::from_pth(weights_path, dtype, &device)?;
let model = BertModel::load(vb, &bert_config)?;
```

**Escape hatch:** `embedder_dtype = "f32"` in `~/.anno-rag/config.toml` reverts to fp32 if a user reports recall issues.

**Verification:** `bench_recall.rs` (§3.1) enforces recall@10 ≥ 95% of `main` (fp32) baseline. PR blocks if it doesn't hold.

## 4. Task decomposition

| Task | Issue | Estimate | Depends on |
| --- | --- | --- | --- |
| **T1** Bench harness (criterion + sysinfo + CI gate + 5 benches) | #029 | 4-6h | — |
| **T2** Lazy-init `OnceCell<Embedder>`, `OnceCell<Detector>` | #023 | 2-3h | T1 (for measurable gain) |
| **T3** Drop `kreuzberg/ocr`, opt-in `--enable-ocr` fork-to-tesseract | #024 | 4-6h | — (parallel with T2) |
| **T4** Single-backend NER (drop StackedNER chain) | #025 | 1-2h | v0.3 #001 merged + T1 |
| **T5** Embedder fp16 + recall regression test | #026 | 2-3h | T1 (baseline) |
| **T6** `anno-rag bench` CLI subcommand + bench doc refresh | doc | 2-3h | T1-T5 |

**Total:** 15-23h dev (~3-4 days focused).

**Merge order:** T1 → T2 → (T3, T5 parallel) → T4 → T6.

**Final PR gate:** T6 PR must show peak RSS < 1.5 GB on the bench corpus, cold-start < 2s, search p95 < 200ms. Intermediate PRs (T2-T5) only fail on >15% criterion regression, not on absolute SLO.

## 5. Risks and mitigations

| Risk | Mitigation |
| --- | --- |
| fp16 degrades recall below 95% baseline | `bench_recall.rs` detects it; fp32 fallback via `embedder_dtype` config |
| System `tesseract` flaky on Windows (PATH, encoding) | Default skip-OCR; explicit `tesseract_path` config field; CI matrix Win+Mac+Linux |
| GLiNER2Fastino slower than StackedNER at runtime | `bench_search.rs` includes detect path; if p95 explodes, restore StackedNER behind feature flag |
| CI gate 15% too strict given GitHub runner variance | Start at 15%, tighten to 10% after 1 week of stable baselines |
| sysinfo RSS semantics differ Win vs Linux | Per-OS baselines; gate per-OS independently |
| `OnceCell` makes panic-on-init harder to debug | All init errors surface through `Result` from accessor; no `unwrap` in init closures |

- **#028 Format coverage** (XLSX/CSV/TSV/JSONL) — separate v0.6 spec

## 7. Acceptance criteria

The v0.5 release ships when **all of**:

1. ✅ `bench_idle_rss` reports < 200 MB peak idle on Linux + macOS + Windows runners.
2. ✅ `bench_cold_start` reports p95 < 2s.
3. ✅ `bench_search` reports p95 < 200ms warm.
4. ✅ `bench_recall` reports ≥ 95% of main fp32 baseline.
5. ✅ Manual run of `anno-rag bench --corpus piighost-test-multi-format` (locally on jamon's 8 GB laptop) shows peak RSS < 1.5 GB after 1 ingest + 5 searches.
6. ✅ CHANGELOG updated; CI bench gate live on `main`.
7. ✅ README documents `--enable-ocr` flag + `tesseract` install instructions.
