# anno-rag v0.4 — HTTP API + GDPR Core Articles

**Date:** 2026-05-13
**Status:** ready for plan
**Goal:** Add an HTTP API mirroring the MCP surface, plus the three GDPR articles a French legal practice cannot ship without: Art. 30 (audit register), Art. 17 (right to erasure), Art. 15 (right of access). Plus TOML config + CLI flags so the binary is operationally tunable.

v0.2 (committed in main as of 2026-05-13 `09502ee2`) shipped MCP. v0.3 (in flight per BACKLOG #001-#005) improves French NER + adds tabular foundations. **v0.4 makes anno-rag operationally complete for an on-prem deployment at a real cabinet.**

---

## 1. What v0.4 adds

- **HTTP API (axum)** — REST mirror of the MCP tool surface, bound `127.0.0.1` by default, bearer-token auth, JSON in/out
- **GDPR Art. 30 audit log** — append-only JSONL with sha256 hash chain + daily HMAC signature; `anno-rag register --since --until --format pdf|csv` exporter
- **GDPR Art. 17 erasure** — `Pipeline::forget(subject)` + `anno-rag forget --subject ... --reason ...` CLI + MCP/HTTP tools
- **GDPR Art. 15 access** — `Pipeline::find_subject(ref)` + CLI/MCP/HTTP exposure
- **TOML config loader** — `~/.anno-rag/config.toml` overrides defaults
- **CLI flags** — `--data-dir`, `--config <path>`, `--passphrase-env <NAME>`
- **MCP `Error::Mcp` variant** — typed error category for transport / handler failures
- **MCP graceful shutdown** — `tokio::select!` against `ctrl_c` to flush audit + close LanceDB cleanly
- Crate version bump: `0.2.0` → `0.4.0` (skip 0.3.0 — that's the parallel branch)

## 3. HTTP API design

### Subcommand

```sh
anno-rag serve [--bind 127.0.0.1:8400] [--tls-cert <path>] [--tls-key <path>]
```

`anno_rag::api::serve(pipeline, cfg, bind)` is the public entry. Reuses the same `Pipeline` as the CLI + MCP.

### Auth

Bearer token. Token sourced via (in order):
1. `ANNO_RAG_HTTP_TOKEN` env var
2. OS keyring entry `anno-rag:http-token`
3. Generated on first start and printed to stderr once

Token is hex-encoded 32 random bytes; verified per request via constant-time compare.

### Routes (REST mirror of MCP)

```
GET    /v1/health                            -> { status, version }
POST   /v1/search                            -> SearchResult
  body: { query: string, top_k?: usize }

POST   /v1/ingest                            -> { ingested: usize, output_dir: string }
  body: { folder: string, recursive?: bool, output_dir?: string }
  Note: long-running; v0.4 keeps it sync. v0.5 adds /v1/jobs.

GET    /v1/documents?folder=...&limit=...    -> Vec<DocumentMeta>
GET    /v1/documents/:doc_id                 -> { metadata, chunks: Vec<...> }

POST   /v1/subjects/find                     -> SubjectMatches
  body: { subject_ref: string }

POST   /v1/subjects/:ref/forget              -> ErasureReceipt
  body: { reason: string }

GET    /v1/subjects/:ref/export?format=json|csv  -> bytes

POST   /v1/pseudonymize                      -> PseudonymizedText
POST   /v1/rehydrate                         -> RehydratedText
POST   /v1/detect                            -> DetectResult

GET    /v1/audit?since=...&until=...&op=...  -> Vec<AuditEvent>
GET    /v1/audit/register?from=...&to=...&format=pdf|csv  -> bytes
```

### Response shape

All POST/GET return JSON. Errors are typed:

```json
{
  "error": {
    "kind": "NotFound" | "Unauthorized" | "BadRequest" | "Internal" | "VaultCorrupted",
    "message": "...",
    "trace_id": "uuid-v7"
  }
}
```

`trace_id` is logged in audit + returned to client for support correlation.

### TLS

v0.4 ships TLS support but does NOT make it mandatory. Default is loopback bind → operator can choose. Once the binary lands on an internet-facing host, `--tls-cert`/`--tls-key` must be set; the binary refuses non-loopback binds without TLS.

### CORS

Off by default. Re-enable via config (`[http] cors_origins = ["https://..."]`). For local-only Cowork-direct-via-MCP, CORS is irrelevant.

## 4. GDPR Art. 30 — audit log

### What goes in the log

Every "regulated operation" emits one JSONL line. Regulated ops:
- `ingest_one` — per document
- `search` — per query
- `rehydrate` — per call (since this surfaces raw PII)
- `forget` — per subject
- `find_subject` — per call
- `export_subject` — per call
- `mcp_initialize` — Cowork attaching
- Vault open / close (process lifecycle)

### Schema (each line)

```json
{
  "ts": "2026-05-13T14:23:11.345Z",
  "op": "search" | "ingest_one" | ...,
  "actor": "cli" | "mcp:<plugin_id>" | "http:<token_id>",
  "subject_tokens": ["PERSON_42", "IBAN_FR_3"],
  "doc_id": "uuid-v7" | null,
  "details": { ... op-specific fields ... },
  "trace_id": "uuid-v7",
  "prev_hash": "sha256:..."
}
```

### Hash chain

Every line's `prev_hash` is `sha256` of the previous line (whole bytes including trailing `\n`). First line has `prev_hash: "sha256:0000…"` (32 zero bytes). On startup, anno-rag verifies the chain from end → start (or from last verified marker); a break refuses to start unless `--audit-force-recover` is passed (and that flag is itself audit-logged once recovery succeeds).

### Daily HMAC signature

At UTC midnight (or first activity after), anno-rag signs the last N lines with HMAC-SHA256 keyed by the OS keyring entry `anno-rag:audit-key`. Signature stored as `audit.jsonl.sig` siblings:

```
~/.anno-rag/audit/
├── audit.jsonl                       # current day, append-only
├── audit.2026-05-12.jsonl.sig        # daily HMAC over the 2026-05-12 segment
├── audit.2026-05-12.jsonl            # rotated previous day
└── register-2026-Q1.{csv,pdf}        # exported register snapshots
```

Rotation at UTC midnight: rename `audit.jsonl` → `audit.<yesterday>.jsonl`, emit `.sig`, start fresh `audit.jsonl` whose first line carries `prev_hash` of the rotated file's last line.

### `register` exporter

```sh
anno-rag register --since 2026-01-01 --until 2026-03-31 --format pdf
```

Generates a CNIL-style Art. 30 record-of-processing report: ops grouped by purpose (search / ingest / rehydrate / forget), aggregated counts, retention rationale, recipients (Cowork as remote LLM), legal basis (operator-configured, default `art_6_1_f` legitimate interest).

PDF format uses a minimal `printpdf`-based template; CSV is straight tabular. JSON dump (`--format jsonl-raw`) is the unaggregated log for power users.

## 5. GDPR Art. 17 — `forget`

### Mechanics

`anno-rag forget --subject <ref> --reason "operator-supplied"`:

1. Resolve `subject_ref` to all `chunks` whose `entities` array contains a token with that subject ID
2. For each matched chunk:
   - `lance.delete("chunk_id = X'...'")` — soft-delete
3. `lance.optimize(OptimizeAction::Prune { older_than: Duration::zero() })` — drop tombstones
4. `lance.optimize(OptimizeAction::Compact { materialize_deletions: true })` — physically rewrite fragments
5. Drop affected LanceDB tags (per the `forget_drops_tagged_versions` policy — required because tagged versions resist Prune)
6. `vault.remove(subject_id)` — remove all token mappings tied to the subject
7. Append `ErasureReceipt { subject, chunks_deleted, vault_entries_removed, lance_version_before, lance_version_after, timestamp, signature }` to audit log
8. Return the receipt

### Receipt format

```json
{
  "subject_ref": "PERSON_42",
  "subject_resolved_canonical": "Marie Dupont",
  "chunks_deleted": 47,
  "vault_entries_removed": 3,
  "lance_version_before": 17,
  "lance_version_after": 21,
  "timestamp": "2026-05-13T14:30:00Z",
  "actor": "cli",
  "reason": "operator-supplied",
  "trace_id": "uuid-v7",
  "signature": "hmac-sha256:..."
}
```

The receipt is the legal artefact the operator hands to the subject. HMAC-signed with `anno-rag:audit-key`.

### Idempotency

Calling `forget` twice on the same subject: second call deletes 0 chunks, 0 vault entries, returns a "no-op" receipt. Both calls audit-logged.

## 6. GDPR Art. 15 — `find_subject`

```sh
anno-rag find-subject --subject "Marie Dupont"
```

Resolves the subject (via fuzzy match against vault entries — case-insensitive contains, plus exact match on pseudonym tokens), returns:

```json
{
  "subject_canonical": "Marie Dupont",
  "subject_token": "PERSON_42",
  "chunks_referencing": 47,
  "documents": [
    { "doc_id": "...", "source_path": ".../contract.pdf", "chunk_indices": [3, 7, 12] },
    ...
  ],
  "first_seen": "2026-04-01T...",
  "last_seen": "2026-05-10T..."
}
```

CLI prints a summary; MCP/HTTP tool returns full JSON. Audit-logged.

## 7. TOML config loader

```toml
# ~/.anno-rag/config.toml (or path passed via --config)

data_dir       = "/var/lib/anno-rag"  # override $HOME/.anno-rag
embed_model    = "intfloat/multilingual-e5-small"
embed_dim      = 384
default_top_k  = 10
chunk_max_chars = 2048
chunk_overlap  = 256

# v0.2 fields:
vector_index_threshold = 1000
ner_warmup_model       = "SemplificaAI/gliner2-multi-v1-onnx"  # v0.3 default
mcp_server_name        = "anno-rag"

# v0.4 new sections:
[audit]
retention_days = 3650  # 10 years for legal practice
sign_daily      = true
legal_basis_default = "art_6_1_f"
controller_name = "Cabinet d'Avocats Dupont & Associés"

[http]
bind             = "127.0.0.1:8400"
cors_origins     = []
tls_cert         = null
tls_key          = null

[forget]
purge_strategy = "shred"  # "shred" = full erasure; "anonymize" = vault-only (no chunk delete)
```

Field-level `#[serde(default = "...")]` keeps backwards compat: a v0.2 config without `[audit]` still loads with defaults.

## 8. CLI flag changes

```sh
anno-rag [--data-dir <path>] [--config <path>] [--passphrase-env <NAME>] <subcommand>
```

- `--data-dir <path>` — overrides `$HOME/.anno-rag` for this invocation
- `--config <path>` — TOML config file path (default `~/.anno-rag/config.toml`)
- `--passphrase-env <NAME>` — env var name to read the vault passphrase from (default `ANNO_RAG_VAULT_PASSPHRASE`)

New subcommands:
- `anno-rag serve` — HTTP server
- `anno-rag register --since --until --format`
- `anno-rag forget --subject --reason`
- `anno-rag find-subject --subject`
- `anno-rag audit verify-chain` — explicit chain check (also auto-runs on startup)

## 9. Acceptance criteria

v0.4 ships when **all** pass:

1. `cargo build --release -p anno-rag` on macOS + Windows + Linux
2. `cargo check --workspace` clean
3. `cargo test -p anno-rag --lib` — v0.1 + v0.2 + new tests pass
4. **E2E for HTTP:**
   ```sh
   ./target/release/anno-rag serve &
   curl -H "Authorization: Bearer $TOKEN" http://localhost:8400/v1/health
   # → { "status": "ok", "version": "0.4.0" }
   curl -X POST -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
       -d '{"query":"résiliation","top_k":3}' http://localhost:8400/v1/search
   # → { "hits": [...] }
   ```
5. **E2E for GDPR Art. 17:**
   - Ingest a fixture containing "Marie Dupont"
   - `anno-rag forget --subject "Marie Dupont" --reason "test"` returns a non-zero `chunks_deleted` and a signed receipt
   - Subsequent `anno-rag search "Marie Dupont"` returns 0 hits (because vault no longer has the token)
   - `~/.anno-rag/audit/audit.jsonl` has a `forget` line with valid `prev_hash`
6. **E2E for Art. 30:**
   - Run 5 operations
   - `anno-rag audit verify-chain` → "ok"
   - `anno-rag register --since 2026-01-01 --until 2030-01-01 --format csv` produces a CSV with ≥ 5 ops
7. **TOML config:**
   - Place `[audit] retention_days = 7300` in `~/.anno-rag/config.toml`
   - Verify `anno-rag --config $PWD/config.toml audit verify-chain` reads it (logs `retention_days = 7300`)

## 11. Decision log

| # | Question | Decision | Rationale |
|---|---|---|---|
| 1 | Scope sizing | HTTP + 3 GDPR articles + TOML config + 2 MCP code-review fixes | "Operationally complete" threshold for cabinet on-prem ship |
| 2 | TLS | Optional; required for non-loopback binds | Allows quick local dev, mandatory once exposed |
| 3 | Auth | Bearer token, OS keyring backed | Same pattern as Vault key |
| 4 | Ingest via HTTP | Synchronous in v0.4 | Long-running but acceptable for v0.4; v0.5 adds `/v1/jobs` for async |
| 5 | Audit chain | sha256 of prev line + daily HMAC sig | Tamper-evident at rest; HMAC verifies provenance |
| 6 | Audit rotation | Daily at UTC midnight | Standard log-management; supports CSV/PDF Art. 30 register |
| 7 | `forget` includes vault delete | YES — full erasure | Otherwise pseudonymized chunks remain reversible; doesn't match Art. 17 spirit |
| 8 | `find_subject` fuzzy match | YES (vault aliases + case-insensitive contains) | Operator may not know exact pseudonym token |
| 9 | TOML config | Additive over `#[serde(default)]` defaults | v0.2 configs continue to parse |
| 10 | Crate version | 0.4.0 (skip 0.3.0) | v0.3 owns its own version on a parallel branch; main can converge once both land |
| 11 | New deps | axum, tower-http (already there for cors), printpdf | Avoid heavy alternatives (warp, actix); axum is workspace-standard |
| 12 | Graceful shutdown | `tokio::select!` on `ctrl_c` + `service.waiting()` | Per code-review I5 |

## 12. Risks

| Risk | Mitigation |
|---|---|
| `lance.optimize(Prune)` doesn't physically delete fast enough → subject re-appears in subsequent search | Add explicit `lance.optimize(Compact)` after Prune; integration-test that `search(subject)` returns 0 hits post-forget |
| HMAC key in keyring leaks → audit signatures faked | Document threat model: full-trust admin is out of scope for v0.4; consider TEE-backed key in v1.x |
| Hash chain breaks → operator can't start the binary | `--audit-force-recover` flag (audit-logged); document recovery procedure |
| HTTP token theft → unauthorized forget | TLS required for non-loopback; token rotation via `anno-rag rotate-token`; rate-limit `forget` to N per hour |
| Register PDF generation fails on edge cases (unicode, large data) | Stick to ASCII-safe printpdf templates; CSV is the primary format, PDF is convenience |
| MCP shutdown handler races vault flush | Use `tokio::sync::Mutex::lock().await` not `try_lock`; let pending ops complete; tested via integration that simulates SIGTERM mid-op |
