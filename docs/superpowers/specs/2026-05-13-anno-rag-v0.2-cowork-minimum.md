# anno-rag v0.2 — Cowork Minimum Spec

**Date:** 2026-05-13
**Status:** ready for implementation plan
**Goal:** Make v0.1 usable from Claude Cowork via MCP, with anno NER reliable (names actually scrubbed).

v0.1 (PR https://github.com/jamon8888/anno/pull/1) shipped the end-to-end loop as a CLI. v0.2 is the **smallest scope that lets a French legal practice actually use it through Claude Cowork**. Everything else (HTTP API, GDPR audit, citation graph, etc.) is explicitly deferred.

---

## 1. What v0.2 adds

- **MCP server** exposed via `anno-rag mcp` subcommand (stdio transport) — Cowork connects to it as a plugin and gets the retrieval surface
- **Warm anno NER** — `warmup` example downloads anno's default ONNX NER model so name detection actually works (v0.1 fell back to pattern+heuristic and missed "Jean Martin")
- **LanceDB vector index** built once chunk count crosses a threshold — flat scan is fine at 3 fixtures but unusable at 10k chunks
- Crate version bump: `0.1.0` → `0.2.0`

That's it. No HTTP. No audit. No GDPR-article surfaces. No tabular review. No 7-crate decomposition.

## 2. What v0.2 explicitly does NOT do

Tracked in the v1 design (`docs/superpowers/specs/2026-05-12-anno-rag-design.md`) for later releases:

- HTTP API (axum) — v0.3
- GDPR Art. 30 audit / Art. 17 forget / Art. 15 find-subject — v0.3 or v0.4
- Hierarchical legal structure typing — v0.5
- Citation graph (ECLI, Cass./CE/CA) — v0.5
- Privilege gating (avocat-client) — v0.5
- Cross-encoder rerank (`camembert-L6-mmarcoFR`) — v1.0
- Tabular review (Harvey/Legora pattern) — v1.1 (plan already written)
- Watch mode / index encryption at rest / multi-tenant — v1.x
- 7-crate decomposition — when `cargo build` exceeds 30s and stays there
- TOML config loader — v0.3
- Folder root preservation in ingest_one — v0.3
- text_hash short-circuit on re-ingest — v0.3
- Batched chunk-embed (memory) — v0.3

## 3. MCP server design

### Subcommand

```sh
anno-rag mcp
```

Stdio transport (per MCP App convention). Cowork plugin config:
```json
{
  "command": "/path/to/anno-rag",
  "args": ["mcp"],
  "env": { "ANNO_RAG_VAULT_PASSPHRASE": "..." }
}
```

The server opens the existing data dir (`~/.anno-rag/`), loads vault + embedder + store + detector identically to the CLI. No new on-disk state.

### Tools exposed (minimum 4)

```
search(query: string, top_k: int = 10) -> Vec<SearchHit>
  Pseudonymize the query → embed → top-K vector search.
  Returns: [{ doc_id, chunk_id, source_path, folder_path, chunk_idx,
              text_pseudo, page?, char_start, char_end, distance }]
  Cowork sends the pseudonymized text to Claude; user can later call
  rehydrate() if needed.

rehydrate(text: string) -> { text, tokens_rehydrated }
  Replace pseudo-tokens (PERSON_1, NIR_3, IBAN_FR_2, …) with original
  values from the local vault. The only place cleartext PII appears
  outside the vault on disk.

detect(text: string) -> Vec<DetectedEntity>
  Dry-run scan — returns entities + categories without replacing.
  Useful for UI previews ("3 PII entities would be scrubbed").

vault_stats() -> { total_mappings, categories: map }
  Diagnostics. Total tokens + per-category breakdown.
```

**Out of scope for v0.2** (would be v0.3+): `ingest_folder` (long-running task, needs progress reporting), `forget`, `find_subject`, `export_subject`, `verify_citations`.

### Implementation note

v0.1 already vendors cloakpipe's `cloakpipe-mcp` crate (`vendor/cloakpipe/crates/cloakpipe-mcp/`) which uses `rmcp 1.6` and exposes 6 tools (`pseudonymize`, `rehydrate`, `detect`, `vault_stats`, `configure`, `session_context`). We REUSE its tool-handler patterns and wire our own server that adds `search()` on top.

The handler invokes our `Pipeline::search()` from v0.1. No new business logic — just the MCP transport boilerplate + the tool definitions.

### Authorization

v0.1 has no auth. v0.2 inherits: anyone with stdio access has full access. Cowork's local plugin model is the security boundary. Documented as "single-user, single-tenant" in the README. Multi-tenant is v0.4+.

## 4. Warm anno NER

### Problem

v0.1's `Detector::new()` calls `anno::StackedNER::default()`. Without an ONNX NER model cached, anno's fallback chain ends at pattern+heuristic, which misses many French names (e.g. "Jean Martin" survived v0.1's e2e because the heuristic doesn't flag bare two-capitalized-word strings in body text). Names leak to the LLM.

### Fix

Extend the `warmup_model` example to also pull anno's `gliner_pii` model (or whichever ONNX NER model `StackedNER::default()` picks at runtime), and document running it before first use.

We do **not** add NER startup to `Pipeline::new()` — that would make CLI startup slow and unpredictable. The `warmup` step stays opt-in but documented prominently.

### Concretely

- `crates/anno-rag/examples/warmup_model.rs` extended:
  - Existing: download `intfloat/multilingual-e5-small`
  - Add: also download whatever anno's default NER backend uses
- Document running order in README: `cargo run --example warmup_model --release` (one-shot, ~600 MB total cache)
- v0.2 e2e test restores the name assertions that v0.1 had to relax (Marie Dupont, Jean Martin, Sophie Bernard, Pierre Lefebvre — these now MUST be scrubbed since gliner_pii is cached)

### Risk

If anno's default backend changes between releases, the warmup misses the new default. Mitigation: warmup tries every backend anno declares as default (BERT-ONNX, NuNER, gliner_pii) and skips ones that don't exist.

## 5. LanceDB vector index

After the first `ingest` run that pushes the chunks table past N rows (default N=1000), call:

```rust
tbl.create_index(&["vector"], Index::IvfHnswSq(IvfHnswSqIndexBuilder::default()))
    .execute().await?
```

Done once. Idempotent — calling on an already-indexed column is a no-op. Index build is heavy (~30-60 s for 10k vectors at 384-dim) so it runs in a `tokio::spawn_blocking` task after ingest completes, not in the request path.

Implementation: `Store::maybe_build_index()` called at the tail of `Pipeline::ingest_folder()` if `chunks.count() >= cfg.vector_index_threshold` (new config field, default 1000) and the index doesn't already exist.

## 6. AnnoRagConfig additions

Add three fields to keep config additive (no breaking changes to v0.1 default deserialization):

```rust
pub struct AnnoRagConfig {
    // ... existing v0.1 fields ...

    /// Threshold (chunk count) at which to build an IVF_HNSW_SQ index
    /// on the vector column. Below this, flat scan is fine.
    #[serde(default = "default_vector_index_threshold")]
    pub vector_index_threshold: usize,

    /// HF Hub model ID for the NER model to pre-warm. None disables.
    /// Default uses anno's default backend.
    #[serde(default)]
    pub ner_warmup_model: Option<String>,

    /// MCP server identification string (advertised on `initialize`).
    #[serde(default = "default_mcp_server_name")]
    pub mcp_server_name: String,
}

fn default_vector_index_threshold() -> usize { 1000 }
fn default_mcp_server_name() -> String { "anno-rag".into() }
```

Existing fields untouched. `Default` impl extended to populate the new ones.

## 7. Acceptance criteria

v0.2 ships when **all** pass on a clean dev machine:

1. `cargo build --release -p anno-rag` succeeds on macOS + Windows
2. `cargo check --workspace` clean
3. `cargo test -p anno-rag --lib` — same 14 unit tests as v0.1, all pass
4. `cargo run --example warmup_model --release` downloads BOTH embedder AND NER model (~600 MB total)
5. `cargo test -p anno-rag --test e2e -- --ignored` — passes WITH the NER-required name assertions (Marie Dupont / Jean Martin / Sophie Bernard / Pierre Lefebvre absent from output `.anon.md` files)
6. `anno-rag mcp < /dev/null` initializes cleanly (writes MCP handshake JSON to stdout, exits when stdin closes)
7. A manual smoke test: start `anno-rag mcp`, send a JSON-RPC `tools/list` request, see the 4 tools (search / rehydrate / detect / vault_stats)
8. `anno-rag mcp` invoked from a Cowork plugin config returns search hits for a real query against an indexed corpus

## 10. Validated decisions

| # | Question | Decision | Rationale |
|---|---|---|---|
| 1 | v0.2 scope | Minimum: MCP + warm NER + vector index | User picked option D (smallest viable) |
| 2 | MCP transport | stdio only (no HTTP) | Cowork plugin uses stdio; HTTP defers to v0.3 |
| 3 | MCP tools | search, rehydrate, detect, vault_stats | Minimum for read-only Cowork integration |
| 4 | Auth | None at MCP layer | Single-user single-tenant; OS file perms is the boundary |
| 5 | NER warmup approach | Extend existing `warmup_model` example | One command, two model families |
| 6 | NER startup in Pipeline::new | NO — opt-in only | Auto-download would surprise users with slow startup |
| 7 | Vector index | `IVF_HNSW_SQ`, threshold=1000, built after ingest in spawn_blocking | Lance recommendation; tunable via config |
| 8 | Index build trigger | Post-ingest tail of `ingest_folder` | Doesn't block ingest; idempotent on retry |
| 9 | Config additions | 3 new fields with serde defaults | Forward-compatible; old TOML configs still load |
| 10 | E2E name assertions | Restore (Marie/Jean/Sophie/Pierre must be scrubbed) | With NER warmed, names ARE detectable |
